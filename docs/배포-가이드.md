# Seoworks Merchant — 배포 및 데이터 전달 가이드

> **마켓(Play Store) 배포 제외** — 직원 기기에 APK 파일로 직접 설치하는 방식

---

## 1. 릴리즈 APK 빌드

### 사전 요건

| 항목 | 경로 / 값 |
|------|----------|
| Flutter SDK | 3.x 이상 |
| 서명 키스토어 | `android/app/upload-keystore.jks` |
| 키 설정 파일 | `android/key.properties` |

`android/key.properties` 내용 (git 제외 파일 — 분실 주의):
```
storePassword=seoworks2026
keyPassword=seoworks2026
keyAlias=upload
storeFile=upload-keystore.jks
```

### 빌드 명령

```bash
# 프로젝트 루트에서 실행
cd merchant_local

# 릴리즈 APK 빌드 (서명 자동 적용)
flutter build apk --release
```

### 빌드 결과물

```
merchant_local/build/app/outputs/flutter-apk/app-release.apk
```

| 항목 | 내용 |
|------|------|
| 현재 버전 | 1.0.0+1 |
| 파일 크기 | 약 81.5 MB |
| 빌드 일시 | 2026-03-31 |

### 버전 올리기

`merchant_local/pubspec.yaml` 수정:
```yaml
version: 1.0.1+2   # 앞: 표시 버전, 뒤: 내부 빌드 번호 (항상 증가)
```

## 2. APK 배포 방법 (파일 전달 후 설치)

### Step 1 — APK 파일 전달

아래 방법 중 편한 것 선택:

| 방법 | 절차 |
|------|------|
| **카카오톡** | PC 카카오톡 → 직원과 1:1 대화 → 파일 첨부 → `app-release.apk` 전송 |
| **Google Drive** | Drive에 APK 업로드 → 링크 공유 (뷰어 권한으로도 다운로드 가능) |
| **USB 직접 복사** | PC에서 스마트폰 연결 → 내부 저장소에 APK 복사 |
| **ADB (개발자)** | `adb install -r app-release.apk` |

### Step 2 — 수신 기기에서 설치

1. **출처를 알 수 없는 앱 허용** 설정 (최초 1회)
   - Android 8.0 이상: `설정 → 앱 → 특별한 앱 접근 권한 → 출처를 알 수 없는 앱 설치`
   - 또는 APK 파일을 열면 자동으로 허용 여부를 묻는 팝업 표시

2. **APK 파일 열기**
   - 카카오톡: 파일 수신 후 "열기" 탭
   - Drive: 다운로드 후 알림 탭 → 열기
   - 파일 관리자: 다운로드 폴더에서 `app-release.apk` 탭

3. **설치 진행**
   - "설치" 버튼 탭
   - 완료 후 "열기" 탭

### 업데이트 설치 (기존 앱 위에 덮어쓰기)

- 동일한 서명 키로 빌드된 APK라면 기존 앱 위에 설치 가능
- 기존 데이터 보존됨 (SQLite DB 유지)
- 설치 전 JSON 백업 권장 (아래 3번 참조)

## 3. 데이터 JSON 파일로 전달하는 방법

앱 내 **설정 화면**에 내보내기/가져오기 기능이 구현되어 있습니다.

### 3-A. 데이터 내보내기 (기존 기기 → JSON 파일)

1. 앱 실행 → 하단 네비게이션 **설정** 탭
2. **데이터 내보내기** 섹션 → **JSON 백업** 버튼 탭
3. 공유 시트 팝업 → 카카오톡 / Google Drive / 이메일 등으로 전송

**내보내기 파일 내용:**

```
merchant_backup.json
├── brands           (브랜드 마스터)
├── products         (상품 모델 마스터)
├── items            (재고 아이템 전체)
├── purchases        (매입 기록)
├── sales            (판매 기록)
├── sale_adjustments (판매 조정 내역)
├── status_logs      (상태 변경 이력)
├── shipments        (배송 기록)
├── inspection_rejections (검수 반려)
├── repairs          (수선 기록)
├── sources          (입고 출처)
├── platform_fee_rules    (플랫폼 수수료 규칙)
├── supplier_returns (반품 기록)
├── order_cancellations   (주문 취소)
├── sample_usages    (샘플 사용)
└── exported_at      (내보낸 시각)
```

### 3-B. 데이터 가져오기 (JSON 파일 → 새 기기)

1. 새 기기에 APK 설치 후 앱 실행
2. 설정 → **데이터 가져오기** → **JSON 가져오기** 버튼 탭
3. 파일 선택 팝업 → `merchant_backup.json` 선택
4. 가져오기 완료 메시지 확인

> **주의:** 가져오기는 기존 데이터에 병합(Insert or Ignore) 방식으로 동작합니다.
> 완전 초기화 후 복원하려면 앱 데이터 초기화 후 가져오기를 실행하세요.

### 3-C. CSV 내보내기 (엑셀 분석용)

설정 → 데이터 내보내기 → **판매 내역 CSV** 또는 **재고 CSV** 버튼

- PC에서 Excel / Google Sheets로 열기 가능
- 인코딩: UTF-8 (Excel은 메모장으로 열어 UTF-8 BOM 저장 후 열기)

## 4. 최초 설치 시 초기 설정

| 순서 | 항목 | 방법 |
|------|------|------|
| 1 | APK 설치 | 위 2번 절차대로 |
| 2 | 앱 실행 | 아이콘 탭 |
| 3 | POIZON API 키 입력 | 설정 → API 설정 → 키 입력 (기기별 개별 입력) |
| 4 | 데이터 복원 | 설정 → JSON 가져오기 (기존 기기에서 백업한 파일) |

> **API 키는 기기 내 보안 저장소(FlutterSecureStorage)에 저장**되며
> APK 파일이나 JSON 백업에는 포함되지 않습니다. 각 기기에서 직접 입력하세요.

## 5. 주요 파일 경로 참조

```
merchant_local/
├── android/
│   ├── key.properties          ← 서명 설정 (git 제외, 분실 금지)
│   └── app/
│       └── upload-keystore.jks ← 서명 키스토어 (git 제외, 분실 금지)
├── build/
│   └── app/outputs/flutter-apk/
│       └── app-release.apk     ← 빌드 결과물
└── pubspec.yaml                ← 버전 관리
```

> `key.properties`와 `upload-keystore.jks`를 분실하면 동일 서명으로 업데이트 APK를 빌드할 수 없습니다.
> **별도 안전한 곳에 백업**해두세요 (USB, Google Drive 비공개 폴더 등).
