# 작업일지

## 2026-03-26 — 코드리뷰 기반 품질 개선

### 수행 내용
DB/UI/API 3개 레이어 전체 코드리뷰 후 검증된 이슈 8건 수정 완료.

### 변경 파일 및 내용

| 파일 | 변경 내용 |
|------|-----------|
| `data_export_service.dart` | CSV Injection 방어 (`_escapeCsv`) + 3개 export 메서드 에러 핸들링(try-catch + 로깅) |
| `sale_adjustment_table.dart` | `saleId` FK에 `onDelete: KeyAction.cascade` 추가 |
| `purchase_table.dart` | `sourceId`에 `.references(Sources, #id, onDelete: KeyAction.setNull)` FK 제약 추가 |
| `app_database.dart` | 스키마 v3→v4: 14개 인덱스 생성 (items.current_status, FK 컬럼들, POIZON 캐시), v2→v3 ALTER TABLE 재실행 안전 처리 (PRAGMA table_info 확인) |
| `poizon_client.dart` | 지수 백오프 재시도 (최대 3회, 500ms/1s/1.5s), 응답 타입 검증 보강 (non-Map 체크) |
| `item_dao.dart` | `validTransitions` 맵 + `updateStatus()`에 상태 전이 검증, `InvalidStatusTransitionException` 추가 |
| `item_detail_screen.dart` | 중복 `_statusLabels` 제거 → `inventory_providers.dart`의 공용 `statusLabels` 재사용 |
