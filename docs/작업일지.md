## 2026-04-04 — Google Drive 동기화 구조 변경 시행착오 및 방향 확정

### 시도한 방법들과 실패 원인

**1차: appDataFolder 스코프 (기존)**
- 문제: OAuth 클라이언트별 격리 → Android/Windows 서로 파일 안 보임

**2차: drive.file 스코프**
- 시도 이유: appDataFolder 대신 일반 폴더 사용하면 공유될 거라 예상
- 실패 원인: drive.file도 동일하게 "해당 클라이언트가 만든 파일만" 접근 가능

**3차: drive (full) 스코프**
- 시도 이유: full access면 클라이언트 무관하게 공유 가능
- 실패 원인: Android GMS가 미검증 앱의 restricted scope 차단 (ApiException: 10)
  - Cloud Console 스코프 등록, 테스트 계정 4명 등록 모두 소용없음
  - Google 앱 심사 통과 없이는 Android에서 사용 불가

**4차: 서비스 계정(Service Account) 방식**
- 시도 이유: OAuth 클라이언트 격리 우회, 로그인 불필요
- 실패 원인 1: GCP 조직 정책(`iam.disableServiceAccountKeyCreation`)으로 키 생성 차단
  - 조직 정책 재정의 권한 없음 → 별도 개인 GCP 프로젝트 생성으로 우회
- 실패 원인 2: 서비스 계정은 개인 Google 계정의 Drive 저장 공간에 파일 생성 불가 (403)
  - "Service Accounts do not have storage quota"
  - Shared Drive는 Google Workspace(유료) 전용 → 사용 불가
  - 폴더 공유 방식도 서비스 계정이 파일 소유자가 되어야 해서 동일 문제

### 확정된 해결 방향
**Desktop OAuth 클라이언트 + drive.file + 브라우저 OAuth (Android/Windows 동일)**
- 같은 OAuth 클라이언트 ID 사용 → 같은 계정 로그인 시 파일 공유 가능
- drive.file은 비제한 스코프 → Google 심사 불필요
- 브라우저 OAuth 사용 → GMS 차단 없음
- 토큰 flutter_secure_storage 저장 → 재시작 시 자동 복원

- `settings_screen.dart`: Galaxy Z Fold 5/6 하단 safe area 패딩 추가
- `AndroidManifest.xml`: 앱 이름 `Seoworks` → `SEOWORKS`

### 현재 코드 상태 (미완성)

## 2026-04-03 — Phase 3-5: SyncEngine 구현

### 수행 내용

**신규 파일: `lib/core/services/sync_engine.dart`**
- `SyncManifest` 모델: schemaVersion / tableHlcs / lastSyncedAt / lastSyncedBy
- `SyncResult` + `SyncStatus` enum
- `SyncEngine.sync()` 전체 플로우:
  1. 로그인 확인
  2. 리모트 `sync_manifest.json` 다운로드 → 스키마 버전 확인
  3. 로컬 max HLC 수집 (`SELECT MAX(hlc) FROM <table>`)
  4. 테이블별 download / upload 대상 결정 (HLC 비교)
  5. 다운로드 + CRDT 머지 (FK 순서대로: brands → sources → … → sample_usages)
  6. 업로드 (hlc != '' 레코드만)
  7. manifest 갱신 + 업로드
  8. SyncMeta `last_sync_at` 저장
- `_mergeRecords()`: LWW (Last-Write-Wins) — 리모트 HLC > 로컬 HLC 시 덮어쓰기
- `_insertRaw()` / `_updateRaw()`: raw SQL `customStatement()`로 타입 독립적 처리
- `_hlcGreaterThan()`: `Hlc.parse()` 비교, 실패 시 문자열 사전순 폴백
- `purgeOldDeletedRecords()`: 90일 이상 soft-deleted 레코드 하드 GC

**`lib/core/providers.dart`**
- `syncEngineProvider`: `SyncEngine(db, driveService, clock)` Provider
- `lastSyncAtProvider`: `FutureProvider<DateTime?>` — SyncMeta `last_sync_at` 읽기

### 주의사항
- `_insertRaw`: `INSERT OR IGNORE` 사용 — 이미 존재하는 레코드는 건드리지 않음
- `_updateRaw`: id 제외한 컬럼만 SET 처리
- `bool → int` 변환: `_toSqlValue()` 헬퍼 — SQLite는 bool 타입 없음
- 최초 sync(manifest 없음): 로컬 데이터 전체 업로드
