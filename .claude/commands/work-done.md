# 작업 완료 처리 (/work-done)

작업 완료 후 문서 업데이트를 일괄 처리한다.

## 실행 순서

1. 사용자에게 완료된 작업 내용을 확인 (없으면 현재 대화에서 추론)
2. `docs/작업일지.md` 하단에 다음 형식으로 추가:

```markdown
## YYYY-MM-DD — [작업 요약]
- [완료 항목 1]
- [완료 항목 2]
```

3. `memory/sprint_status.md`에서:
   - 완료된 항목을 이력으로 이동
   - P1/P2/P3 우선순위 업데이트

4. 완료 요약 출력 (1-3줄)

## 규칙
- CLAUDE.md는 수정하지 않는다 (세션 상태는 memory에서 관리)
- 커밋은 하지 않는다 (명시적 요청 시에만)
